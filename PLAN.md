# 구현 단계 계획

## 0단계. 프로젝트 기본 설정

목표:
- Stitch export 파일을 React 프로젝트로 옮기기 전 기본 개발 환경을 구성한다.

작업:
- GitHub 저장소 생성
- React 프로젝트 생성
- `_light` 폴더 기준 파일 확인
- 기본 폴더 구조 설정
- 라우팅 구조 설정
- Firebase 사용 여부 확정
- 환경변수 파일 구조 설정
- 초기 커밋 생성

결과물:
- 실행 가능한 React 기본 프로젝트
- GitHub 버전 관리 시작
- 개발 기준 폴더 구조 확정

---

## 1단계. 기준 파일 정리

목표:
- `_light` 폴더만 메인 UI 기준으로 사용한다.

작업:
- `landing_page_light` → 메인 페이지
- `scanner_modal_light` → 스캔 페이지
- `admin_dashboard_light` → 관리자 대시보드
- `manage_poster_form_light` → 포스터 등록/수정 페이지
- 불필요한 dark 버전 제외
- 공통 UI 요소 확인

---

## 2단계. UI 문구 한국어화

목표:
- 영어 문구를 부드러운 한국어 말투로 수정한다.

작업:
- 사용자 페이지 문구 수정
- 관리자 페이지 문구 수정
- 버튼/라벨/상태값 한국어화
- 미등록 포스터 안내 문구 수정
- 한국어 테스트용 더미 데이터 추가

예시:
- Start Scanning → 스캔을 시작해볼까요?
- Point your camera at a poster → 포스터를 카메라에 비춰주세요
- Unregistered Poster → 등록되지 않은 포스터예요
- Retry Scan → 다시 스캔하기
- Exit → 나가기
- Add Poster → 포스터 등록하기
- Save Poster → 저장하기
- Public → 공개
- Private → 비공개

---

## 3단계. React 프로젝트 구조화

목표:
- Stitch UI를 React 컴포넌트 기반 구조로 정리한다.

작업:
- 페이지 라우팅 설정
- 공통 레이아웃 분리
- 공통 버튼/카드/입력창 컴포넌트 분리
- CSS 또는 Tailwind 구조 정리

예상 라우트:
- `/`
- `/scan`
- `/login`
- `/admin`
- `/admin/posters/new`
- `/admin/posters/:id/edit`

---

## 4단계. 사용자 인증 구조 구현

목표:
- 모든 사용자가 일반 사용자이면서, 로그인 시 관리자가 될 수 있는 구조를 만든다.

작업:
- 회원가입 또는 로그인 방식 결정
- 로그인 페이지 구현
- 로그인 상태 관리
- 로그인한 사용자만 관리자 페이지 접근 가능
- 로그아웃 기능 구현

주의:
- 여기서 “관리자”는 고정된 1명이 아니라, 로그인 후 포스터를 관리할 수 있는 사용자로 정의한다.

---

## 5단계. Firebase 연동

목표:
- 클라우드 기반 데이터 저장 구조를 구현한다.

작업:
- Firebase 프로젝트 생성
- Firebase Authentication 연결
- Firestore 연결
- Firebase Storage 연결
- 환경변수 설정
- 보안 규칙 기본 설계

저장 데이터:

- 공개/비공개 상태

## 6단계. 포스터 관리 기능 구현

목표:
- 로그인한 사용자가 포스터를 등록/수정/삭제할 수 있도록 구현한다.

작업:
- 포스터 등록
- 포스터 수정
- 포스터 삭제
- 포스터 목록 조회

- Spotify 링크 입력 검증
- 한국어 테스트용 더미 데이터 표시

## 7단계. 모바일 스캔 페이지 구현

목표:
- 사용자가 모바일에서 카메라를 열고 포스터를 스캔할 수 있도록 구현한다.

작업:

## 8단계. 이미지 인식 기능 구현

목표:
- 등록된 포스터 이미지를 인식하여 해당 포스터 데이터를 찾는다.

작업:
- WebAR 또는 이미지 인식 라이브러리 연결
- 등록된 포스터 기준 이미지 데이터 생성
- 인식 성공 시 포스터 데이터 조회
- 비공개 포스터는 무반응 처리
- 미등록 포스터는 안내 모달 표시
